THE SHORT VERSION

Local models become useful when they can check reality.

ZyleBot is a local-first agentic chat app: a vanilla browser interface talks to FastAPI, FastAPI talks to the model loaded in LM Studio, and a small registered toolset lets that model inspect files, query the system, use the web, and propose controlled actions.

Inference, conversations, file access, and speech transcription stay on your machine. Web search, page fetching, and weather contact external services only when you use them. No hosted inference account or default-provider API key is required.

WHAT YOU GET

More than a one-shot chat window.

  • Agent loop Reason, act, observe, and correct course.
  • Live execution Stream reasoning, tools, approvals, and answers.
  • Local model control Detect, label, monitor, and switch LM Studio models.
  • Durable context Keep independent conversations in SQLite.
  • Voice + vision Dictate locally or attach an image for inspection.

QUICK START // 01

Install the app.

RUNTIME

Python 3.14

Use an isolated virtual environment for the pinned dependencies.

INFERENCE

LM Studio

Download a chat model that reliably emits OpenAI-compatible tool calls.

PLATFORM

Windows / macOS / Linux

Windows is the primary test target; native shell paths exist for all three.

Windows // PowerShell
py -3.14 -m venv venv
.\venv\Scripts\python.exe -m pip install -r requirements.txt
Copy-Item .env.example .env
macOS / Linux
python3.14 -m venv venv
./venv/bin/python -m pip install -r requirements.txt
cp .env.example .env

The copied .env works with LM Studio's standard local endpoint at localhost:1234. Change it only when you need a different model, user name, timeout, database path, or tool limit.

QUICK START // 02

Load a model and start the local server.

Use LM Studio's Developer → Local Server screen, or its CLI. The in-app start control also calls lms server start when the backend can reach the CLI but cannot reach the server.

LM Studio CLI
lms server start
lms load <model-id>
GPU

Maximum offload

Keep all model layers on the GPU when memory allows.

CONTEXT

Start at 65,536

A practical 64k target for a 12 GB card before shared-memory spill.

ATTENTION

Flash Attention on

Use LM Studio's optimized attention path when the model supports it.

ZyleBot detects the loaded context-window size and turns it into the live usage gauge in the header. If model memory spills into shared GPU memory, inference can slow down sharply.

QUICK START // 03

Run the FastAPI app.

Project root // PowerShell
.\venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload

Open 127.0.0.1:8000. The interactive API reference lives at /docs. There is no Node.js dependency or frontend build command—HTML and CSS changes need only a refresh.

TOOL ARSENAL // SAFE TIER

Read-only evidence gathering runs automatically.

These tools let the model inspect reality without intentionally changing local state. The loop can chain them without stopping for an approval card.

ToolPurpose
list_directoryList contents within the ZyleBot project folder only, with basic metadata.
read_fileRead a text file up to the configured byte limit, with best-effort UTF-8 decoding.
search_filesFind files recursively with a glob pattern.
get_file_infoInspect file or folder metadata.
get_system_infoReport OS, CPU, memory, and per-drive disk usage.
web_searchSearch through DuckDuckGo by default.
fetch_urlExtract readable page text in offset-based chunks.
get_weatherResolve a place and return current Open-Meteo conditions.

TOOL ARSENAL // CONTROLLED ACTIONS

Changes require authority. Shell commands require classification.

write_fileCreate or overwrite a UTF-8 text file after approval.
append_fileAppend text, creating the file when needed, after approval.
make_directoryCreate a directory and missing parents after approval.
run_commandInvoke PowerShell on Windows or the configured shell on macOS/Linux, subject to the command guard.
ALLOW

Known read-only

A narrow allowlist can continue without interrupting the turn.

CONFIRM

Not proven safe

The exact command pauses in an Approve / Deny card.

BLOCK

Clearly dangerous

Destructive, elevated, encoded, or nested-shell patterns are refused.

The command guard is a fail-closed safety layer, not a sandbox. An approved command runs with the ZyleBot process's operating-system privileges—approve only what you would run yourself.

HOW IT WORKS // ONE MESSAGE

From request to grounded answer.

  1. You send a messageThe browser posts text and, when attached, an image to the local FastAPI server.
  2. ZyleBot rebuilds the contextThat conversation's saved messages go to the model loaded in LM Studio.
  3. The model answers or requests a toolTool requests use registered names and validated structured arguments.
  4. The boundary is enforcedRead-only work continues, consequential action pauses, and blocked commands are refused.
  5. Evidence returns to the modelIt can continue, correct course, choose another tool, or finish.
  6. The browser receives a live streamReasoning, tool events, approval cards, and answer text arrive over SSE.
  7. The conversation is savedSQLite keeps it independent and restores it after a restart.

The loop stops after AGENT_MAX_STEPS tool rounds. Repeated identical calls are skipped, and reaching the ceiling forces an answer from the evidence already gathered.

MODEL CONTROL

Runtime state wins over stale configuration.

models.json maps LM Studio IDs to friendly aliases and preferred context lengths. ZyleBot reads it fresh and aligns itself to the model LM Studio actually has loaded at startup.

A switch from the header unloads the current model first, keeping a single-GPU machine from accidentally holding two models in VRAM.

MEMORY

The app remembers; the model stays stateless.

Conversations and messages live in data/zylebot.db by default. Starting a new chat creates a clean context; reopening one rebuilds its history.

The optional USER_NAME value is the only fact intentionally shared across every conversation.

PRIVACY + NETWORK

Local-first, stated precisely.

Inference, the agent loop, stored conversations, file tools, and default speech transcription run locally. ZyleBot needs no hosted-inference account.

  • Search contacts the configured provider.
  • Fetch contacts the requested page.
  • Weather contacts Open-Meteo.
  • First voice use downloads the selected Whisper model, then uses its cache.

KNOWN LIMITS

Boundaries worth knowing.

  • Vision runs without tools because the current stack drops images when tools are attached.
  • Web extraction can fail on bot-protected or JavaScript-heavy sites.
  • Shell execution is controlled but not sandboxed.
  • Large contexts may spill into shared GPU memory and slow sharply.
  • Platform coverage is cross-platform, with the most testing on Windows.
