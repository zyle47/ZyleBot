Tool Arsenal

A registry of tools the model can reach for — filesystem, system, web — split into what runs automatically and what waits for you.

Read-only tools run automatically

list_directory, read_file, search_files, get_file_info, and get_system_info inspect your machine without changing anything. web_search, fetch_url, and get_weather reach outward — keyless, via DuckDuckGo and Open-Meteo. These are the SAFE tier: no prompt, no delay.

Action tools wait for a human

write_file, append_file, make_directory, and run_command can change your system, so they sit in the CONFIRM_REQUIRED tier. Calling any of them pauses the agent loop and raises the approval card — see Approval Gate.

One registry, model-agnostic

Tools are declared once and exposed to whichever model is loaded in LM Studio — switching models (see Model Control) doesn't change what's available, only how well the model uses it.
