PRODUCT // 01 // THE ENGINE

Agent Loop

A chatbot gives you an answer. ZyleBot keeps working until it has one worth giving.

Your request becomes a live cycle of reasoning, tool use, observation, and correction. ZyleBot can inspect what is real, gather what is missing, and turn the evidence into a useful result — while you watch the work happen on your own machine.

Multi-step execution Live streaming Persistent memory Human control

WHY IT EXISTS

Because the first answer is often only the beginning.

Real questions rarely fit inside one model response. The model may need to find a file, read a page, check the weather, compare results, discover that its first idea was wrong, and try a better path. A one-shot chat can describe that work. The Agent Loop can actually do it.

That difference turns a local language model from a clever text generator into a practical operator: still fallible, still visible, but able to move through a task instead of stopping at a suggestion.

THE CYCLE

One request. As many useful steps as the job needs.

  1. Understand the goal ZyleBot reads the current message together with the relevant conversation history.
  2. Choose the next move The model decides whether it can answer or needs evidence from an available tool.
  3. Act and observe A safe tool runs automatically, or a system-changing action pauses for your approval.
  4. Re-evaluate The result goes back into the loop, where the model can continue, change course, or finish.
  5. Deliver the result The final response is grounded in what the loop found and did, not merely what the model remembers.

This is the ReAct pattern in practical form: reason, act, observe, repeat. The limit is configurable through AGENT_MAX_STEPS, so the loop has room to solve a task without being allowed to wander forever.

VISIBLE BY DEFAULT

Watch the work, not a spinner.

Reasoning, tool activity, and the final answer arrive over a streaming connection as they happen. The model's reasoning is kept separate from its answer and rendered in a dimmed, collapsible block. Open it when you want the trail; close it when you only want the result.

Tool calls appear as their own events, so a slow web lookup does not look like a frozen app. You can see whether ZyleBot is searching, reading, checking, or waiting on you.

BUILT TO FINISH

Guardrails against useless loops.

Agentic does not have to mean endless. Identical tool calls are recognized and skipped. Failed or empty paths are not supposed to be hammered repeatedly. If the configured step ceiling is reached, ZyleBot asks the model to stop calling tools and produce the best answer supported by what it already gathered.

And when a local model returns an empty final message after doing real work, the loop has fallback handling so the conversation does not end in a blank bubble.

A REAL-WORLD EXAMPLE

“Find the relevant file and explain what this project does.”

01 search the project tree 02 identify README and configuration 03 read the useful sections 04 connect the evidence 05 answer in plain language

The valuable part is not that the model can write a convincing project summary. It is that it can inspect your current project before writing one. The same pattern applies to researching a current topic, checking a machine, reading a long document in chunks, or collecting the facts needed for a decision.

CONTINUITY

Each conversation becomes its own working context.

Conversations live in SQLite and survive a restart. When you return, ZyleBot rebuilds the model context from that conversation's saved messages rather than relying on invisible state inside LM Studio. Start a new chat and you get a clean context; reopen an old one and its history is there.

This makes the mental model simple: the app owns the memory, the local model remains stateless, and every conversation is an independent workspace. The context gauge in the header shows how much of the loaded model's window that workspace is using.

SPEAK NATURALLY

Voice without renting a cloud.

Use the microphone when typing is the slow part. Speech is transcribed locally with faster-whisper on the CPU, leaving precious GPU memory available for the language model. The transcription enters the same conversation and the same Agent Loop as typed input.

SHOW, DON'T DESCRIBE

Vision when words are not enough.

Paste or attach an image and ZyleBot can inspect it in a dedicated vision turn. Image turns intentionally run without tools because the current local-model stack cannot reliably receive both at once. Older images collapse to lightweight placeholders on later turns, keeping the chat usable without repeatedly spending vision tokens.

YOU REMAIN THE OPERATOR

Autonomy for research. Permission for impact.

The loop can move quickly through read-only work: inspect files, query system information, search the web, fetch a page, check weather, and continue reasoning. The moment it proposes an action that changes files or invokes the shell, the loop stops at the Approval Gate.

That pause is part of the engine, not a warning pasted on top. Approve and the exact call runs once. Deny and nothing runs; the model receives the refusal and can offer another route. ZyleBot gets enough agency to be useful without quietly borrowing your authority.
