Model Control

Swap the loaded model from the header dropdown — no config edit, no restart.

Switching without leaving the chat

The model dropdown drives the lms CLI directly, loading and unloading models in LM Studio on your behalf. Aliases and per-model context limits live in models.json — pick a friendly name, ZyleBot resolves the real model id underneath.

The context gauge

The header bar tracks real token usage against the loaded model's context window, turning amber at 70% and red at 90%. It's read from contextUsed on every poll, so a routine health check never wipes it mid-conversation.

Start the server from the app

If LM Studio's server isn't running, the header shows a Start server button that runs lms server start for you — one less window to keep open.
